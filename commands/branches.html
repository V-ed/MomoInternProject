Commandes pour gérer les branches

Vous trouverez dans cette section des commandes utiles pour gérer les branches. Étant un concepte qui caractérise git, ces commandes en sont d'une assez grande utilitée.

git branch

Cette commande est celle qui gère toutes la gestion des branches dans un repository git local. Ceci inclut leur création, leur suppression, renommer une branche et voir toutes les branches (locales et externes).

Voici une liste d'exemples de commandes pour gérer les branches dans un repository :

  • git branch test : Créée une nouvelle branche nommée test à partir de la branche que nous sommes présentement;
  • git branch test2 test : Créée une nouvelle branche nommée test2 étant une copie de la branche test;
  • git branch test 62c30a : Créée une nouvelle branche nommée test à partir du commit ayant l'identifiant (compressé) 62c30a;
  • git branch -d test : Supprime une branche nommée test (-d pour "delete");
  • git branch -m ajout-tests : Renomme la branche que nous sommes présentement pour ajout-tests;
  • git branch -m test ajout-tests : Renomme la branche nommée test pour ajout-tests;
  • git branch -a : Affiche une liste de toutes les branches existantes, autant locales que externes;
  • git branch --list : Affiche une liste de toutes les branches locales;
  • git branch -r : Affiche une liste de toutes les branches externes;

Alors que tous ces exemples gèrent les branches localement, quelques opérations doivent être fait à travers d'autres commandes : par exemple, pour envoyer une branche sur le repository distant, il faut utiliser git push (cliquez sur la commande pour en savoir plus) et pour aller sur une nouvelle branche, il faut utiliser git checkout, expliquée juste en bas.

git checkout

Cette commande sert à se promener de branches en branches. C'est à travers cette commande que les fichiers / dossiers seront modifiés pour refléter les modifications effectuées dans la branche à checkout.

Voici quelques exemples d'utilisation de cette commande :

  • git checkout test : Prend les modifications de la branche test dans le dossier local et considère le projet étant maintenant sur cette branche;
  • git checkout test -f : Exactement comme la commande précédante, mais exécute le changement de branche même si des modifications sont présentes dans le workspace et / ou dans l'index;
  • Spécial : git checkout -b test2 : Cette commande fait deux choses en même temps : elle créée la branche test2 (un peu comme si on aurait fait git branch test2!) et exécute ensuite un git checkout test2 automatiquement! Cela veut donc dire qu'avec l'option -b, la branche n'a pas besoin d'exister pour pouvoir la checkout!

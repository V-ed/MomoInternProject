Commandes pour gérer le repository

Vous trouverez dans cette section des commandes utiles pour gérer un repository git. En passant par sa création jusqu'à la visualisation des commits dans le repository, savoir où on est rendus dans notre projet est la spécialité de ces commandes.

git init

Cette commande sert à initialiser le repository local. En effet, on ne l'utilise qu'au début, et son utilitée après la création du repository est plutôt nulle.

Bref, cette commande est le début d'un projet!

En savoir plus

git clone

Cette commande sert à cloner un repository vers un dossier local. En disant "cloner" ici, on réfère au fait que l'on télécharge* une copie d'un repository externe sur sa machine. Donc, faire un git clone est prendre un repository déjà existant.
Bref, cette commande sert à accéder à un projet déjà en cours!

* un téléchargement n'est pas obligatoire cependant : on peut cloner un repository déjà sur notre machine!

Ainsi, il serait parfois préférable d'ajouter des options pour ne voir qu'un résumé des commits et un status visuel pour mieux visualiser les branches : git log --graph --oneline --all.

git diff

Cette commande permet de savoir les modifications qui sont apportées au code en affichant les lignes enlevées et les lignes ajoutées. En fait, il est important de savoir qu'une ligne modifiée, pour git, est une ligne qui est enlevée et ajoutée à nouveau. Cette commande ressemble beaucoup à la commande git status, que vous pouvez lire plus bas dans la page.

Bref, elle sert à savoir quelles différences sont présentes entre le dernier commit et les modifications courantes!

En savoir plus

git remote

Cette commande sert à gérer l'association des branches locales avec celles du repository distant. Sans arguments, elle affiche les liens externes, tout simplement. Cependant, sa vraie puissance provient des arguments qu'elle accepte, tels que git remote add [url], git remote rename [old] [new] ou git remote rm [url].

Bref, cette commande gère les liens entre les branches locales et celle sur le repository distant, ce qui signifie aussi qu'une branche locale peut être nommée différemment que celles sur le repository distant et tout de même les liés ensemble!

En savoir plus

git status

Cette commande est un peu comme la commande git diff dans le sens où l'on peut savoir avec elle quelles modifications sont apportés au code entre le dernier commit et les modifications locales. Cependant, celle-ci n'affiche pas le contenu des modifications, mais plutôt les fichiers (et leur chemins d'accès relatif à la racine du repository) qui contiennent des changements.

Bref, elle sert à voir le status des fichiers prêt à être mis dans l'index pour être ensuite mis dans un commit.

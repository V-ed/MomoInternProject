Les services externes

Introduction

Les services externes sont des services permettant d'héberger des repositories et d'ajouter des fonctionnalités reliées aux projets git pour permettre aux équipes de mieux travailler en équipe et de mieux performer.

On pourrait penser à un projet publique que tout le monde peut contribuer (communément appelé open source, ou open sourced project) : cette facilité à contribuer est due aux services externes, permettant de tels projets d'être partagés.

Voici une liste de quelques services externes les plus populaires, mais beaucoup plus existes!

GitHub

GitHub est le service que je préfère d'entre tous : il est le plus reconnu pour accueillir les projets open sourced (code visible à tout le monde) et en plus d'être rapide, il est complet et ajoute toutes les fonctionnalités pour avoir un projet qui va vers l'avant. Gratuit pour tout les projets open source et payant pour les projets privés, il offre entre autres :

  • Un système pour décrire facilement le but du projet (un fichier README.md dans la racine du projet);
  • Un système de gestion des problèmes / demandes clients, soit les issues :
    • Un endroit pour discuter des changements et de la faisabilité, par exemple;
    • Un système d'assignation de tâches;
    • Une attribution de catégories (Labels) pour mieux trier les issues;
    • ☆ Une intégration intelligente avec les commits pour pouvoir modifier les status des issues selon le message des commits;
  • Un système de gestion des demande de modifications (les pull requests) :
    • Une association inter-repository, en d'autres mots l'autorisation de modifications par des personnes externes au projet tout en laissant à l'équipe la possibilité d'accepter ou refuser les modifications;
    • Un autre endroit pour discuter de l'implémentation d'une nouvelle fonctionnalité, par exemple;
  • L'intégration de systèmes tierces pour, par exemple, exécuter des tests automatiquement à chaques commits.

Il offre de plus aux étudiants un "Student Pack" pour accéder à plusieurs produits utiles pour l'apprentissage de l'informatique, ainsi que d'autres produits tel qu'un nom de domaine gratuit grâces à des services tierces partenaires à GitHub.

Il faut bien faire la différence entre Git et GitHub cependant : GitHub n'est pas Git, tout comme Javascript n'est pas Java!

BitBucket

BitBucket est une solution pour les petites équipes pour avoir des repository externes privés gratuits. En effet, il permet aux équipes de 5 personnes et moins de pouvoir utiliser la puissance de git sans avoir à afficher son code au monde entier. Il offre la plupart des points mentionnés pour GitHub ci-haut lorsque combiné avec une autre produit de Atlassian (la compagnie qui a créé BitBucket) : Jira. Jira sert à toute la logistiques des issues, discussions, etc. tandis que BitBucket accepte le repository. La version payante ne coûte pas trop cher et il y a même l'option de rouler une instance de BitBucket sur les serveurs même de votre équipe, ce qui est très pratique pour le niveau sécurité de toute entreprises.

GitLab

GitLab est probablement le plus tentant pour les programmeurs indépendants puisqu'il offre la possibilité de créer sans limite des repositories privés et ainsi cacher son code au public tout en laissant la puissance de Git à être accessible. Il offre encore une fois comme GitHub un service d'issues et de pull requests, et contient de plus son propre service de tests automatisés / préparation à compiler le projet pour une nouvelle mise à jour. Comme BitBucket, il est possible de rouler une instance de GitLab dans les serveurs des équipes mêmes, ainsi, la sécurité est aussi bien accrue de cette manière.

VSTS - Visual Studio Team Services

Visual Studio Team Services est le service tierce de Microsoft pour gérer les repositories externes en ajoutant leurs utilités. Il permet aux équipes d'utiliser la technique Agile ou Scrum selons leurs désirs et permet un contrôle granuleux des permissions de chaque individu. Comme BitBucket, VSTS permet d'avoir un nombre illimité de repository privé tant que le nombre de personnes y ayant accès est moins de 5.

Conclusion

Bref, une tonne de services existe, chacun avec leurs propres différences et leurs propres fonctionnalités / design, et il est important que lorsque l'équipe planifie le travail qu'elle va accomplir, une discussion sur la destination du repository soit bien étudiée selon leurs besoins - c'est après tout le service qui contiendra tout votre code!
